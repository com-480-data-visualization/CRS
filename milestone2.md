## Weight

## Written Deliverable

Submit two A4 pages describing the project goal.

The document should include:

- Sketches of the visualization you want to make in the final product.
- The tools you will use for each visualization.
- The past or future lectures you will need for each visualization.
- A breakdown of the project goal into independent implementation pieces.
- A core visualization, or minimum viable product, that must be completed by the end.
- Extra ideas that are more creative or challenging, but could be dropped without
  endangering the meaning of the project.

## Prototype Review

You should have an initial website running with the basic skeleton of the
visualization and widgets.

## Practical Checklist

- Define the project goal clearly.
- Prepare rough sketches of the planned visualizations.
- Decide which tools or libraries each visualization will use.
- Map each visualization to relevant course lectures.
- Split the implementation into independent tasks.
- Mark the minimum viable visualization separately from stretch features.
- Have a basic website prototype running.
- Include placeholder visualization/widget structure, even if the final
  interactions are not finished yet.
